Architekturansätze für das Whiteboard

Drei Wege zum Ziel — jeder mit unterschiedlichen Trade-offs

A

tldraw-basiert (Empfohlen)

Canvas-Engine: tldraw — Open-Source Whiteboard-Engine (React/TypeScript). Bringt Stift, Formen, Text, Radierer, Zoom, Undo/Redo out-of-the-box mit.

Realtime-Sync: tldraw hat ein eingebautes Sync-Protokoll basierend auf Y.js (CRDT). Konfliktfreie Synchronisation auch bei gleichzeitigem Zeichnen.

Persistenz: Zustand wird als JSON-Dokument gespeichert → PostgreSQL.

Vorteile: Extrem viel out-of-the-box, aktive Community, Apple Pencil Support, erweiterbar.

Nachteile: Große Dependency (~500KB), eigene Opinions über UX die evtl. angepasst werden müssen.

Aufwand MVP: ★★☆☆☆ (mittel-niedrig)

B

Excalidraw-basiert

Canvas-Engine: Excalidraw — ebenfalls Open-Source, bekannt für den "Handgezeichnet"-Stil. React-basiert.

Realtime-Sync: Eingebaute Collaboration via WebSocket. End-to-End encrypted möglich.

Persistenz: JSON-basierter Zustand, gut speicherbar.

Vorteile: Passt perfekt zum Handschrift-Stil den du gewählt hast! Intuitiv, beliebt, gute Touch-Unterstützung.

Nachteile: Weniger flexibel bei Custom-Hintergründen (kariert, Koordinatensystem). Stiftdruck-Support eingeschränkt.

Aufwand MVP: ★★☆☆☆ (mittel-niedrig)

C

Custom Canvas + perfect-freehand

Canvas-Engine: Eigene Lösung mit HTML Canvas API + perfect-freehand Library für natürliche Striche.

Realtime-Sync: Eigene WebSocket-Implementierung mit Y.js für CRDT-basierte Synchronisation.

Persistenz: Eigenes Datenmodell in PostgreSQL.

Vorteile: Volle Kontrolle über jeden Aspekt. Perfekt anpassbar an Schulkontext (Hintergründe, Werkzeuge).

Nachteile: Deutlich mehr Entwicklungsaufwand. Alles selbst bauen: Undo/Redo, Formen, Selection, Touch-Handling.

Aufwand MVP: ★★★★☆ (hoch)

Meine Empfehlung: Option A (tldraw)

Warum tldraw?

  • Beste Balance aus Features und Anpassbarkeit
  • Seitenbasiertes Layout umsetzbar
  • Custom Hintergründe (kariert, Koordinatensystem) über Background-Layer
  • Eingebautes Sync-Protokoll
  • PDF-Export über SVG → PDF Konvertierung möglich
  • Aktiv maintained, gute TypeScript-Typen
  • Apple Pencil Druckempfindlichkeit wird unterstützt

Bedenken

  • Bundle-Größe (aber akzeptabel für eine App)
  • Lernkurve für tldraw's interne APIs
  • Einige Custom-Features erfordern tiefes Verständnis der Engine
